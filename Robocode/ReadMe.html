O QUE É ROBOCODE?

Robocode é um jogo de programação onde o objetivo é programar um tanque de batalha robô para competir contra outros robôs em uma arena de batalha. O nome Robocode vem de "código de robô". O jogador é o programador do robô, que não terá influência direta no jogo. Em vez disso, o jogador deve escrever a IA do robô dizendo como ele deve se comportar e reagir aos eventos que ocorrem na arena. As batalhas acontecem em tempo real e são exibidas na tela.

O lema do Robocode é: Construa o melhor, destrua o resto!

Além de ser um jogo de programação, o Robocode é usado para aprender a programar principalmente na linguagem Java, mas outras linguagens como Kotlin e Scala também são possíveis.

Escolas e universidaSdes usam o Robocode como parte do ensino de programação, além de estudar inteligência artificial (IA). O conceito do Robocode é fácil de entender e uma forma divertida de aprender a programar.

O Robocode oferece um ambiente de desenvolvimento completo e vem com um instalador, editor de robôs embutido e compilador Java. O Robocode só exige que um JRE (Java Runtime Environment) já esteja instalado no sistema. Assim, tudo que um desenvolvedor Robocode (Robocoder) precisa para começar já está incluído no arquivo principal de distribuição (robocode-xxx-setup.jar). O Robocode também suporta o desenvolvimento de robôs usando IDEs externas como:

  • IntelliJ IDEA
  • Eclipse
  • NetBeans,
  • Visual Studio Code

Uma IDE externa auxilia muito mais o desenvolvedor do que o editor de robôs embutido no Robocode, que serve apenas para pequenos ajustes rápidos.

O fato do Robocode rodar na plataforma Java permite que ele funcione na maioria dos sistemas operacionais que suportam Java, como Windows, Linux, macOS e outros UNIX. Note que o Java 8 é a versão mínima exigida, mas versões mais recentes também podem ser usadas. Veja Requisitos do Sistema para mais informações.

Saiba que muitos usuários do Robocode (também chamados de Robocoders) acham o Robocode muito divertido, mas também viciante. :-)

O Robocode é gratuito e desenvolvido como um projeto de lazer, sem fins lucrativos. Os desenvolvedores trabalham no Robocode porque acham divertido.

Robocode é um projeto Open Source, ou seja, todo o código-fonte é aberto para todos.

O Robocode é fornecido sob os termos da EPL (Eclipse Public License).

HISTÓRIA DO ROBOCODE

O jogo Robocode foi criado originalmente por Mathew A. Nelson como um projeto pessoal no final de 2000 e se tornou profissional quando ele o levou para seu trabalho na IBM, como um download AlphaWorks, em julho de 2001.

A IBM se interessou pelo Robocode, pois viu uma oportunidade de promovê-lo como uma forma divertida de aprender Java, além de promover a própria linguagem.

A inspiração para criar o Robocode veio do Robot Battle, um jogo de programação criado por Brad Schick em 1994. O Robot Battle, por sua vez, foi inspirado pelo RobotWar, um jogo para Apple II+ do início dos anos 1980.

Artigos da IBM ("Rock 'em, sock 'em Robocode", "Robocode Rumble" e "Secrets from the Robocode masters") e a comunidade do Robocode por trás da RoboWiki tornaram o Robocode muito popular como jogo de programação, sendo usado por muitos anos para educação e pesquisa em escolas e universidades ao redor do mundo.

No início de 2005, Mathew convenceu a IBM a liberar o Robocode como Open Source no SourceForge. Nesse ponto, o desenvolvimento do Robocode havia parado. A comunidade começou a desenvolver suas próprias versões com correções e novos recursos, como "Contributions for Open Source Robocode" e, depois, os projetos RobocodeNG e Robocode 2006, de Flemming N. Larsen.

Eventualmente, Flemming assumiu o projeto Robocode no SourceForge como administrador e desenvolvedor em julho de 2006 para continuar o desenvolvimento do jogo original. Assim, o projeto RobocodeNG foi abandonado e o Robocode 2006 foi fundido na nova versão oficial 1.1, com várias melhorias. Desde então, muitas versões novas foram lançadas com mais recursos e contribuições da comunidade.

Em maio de 2007, o cliente RoboRumble foi incorporado ao Robocode. O RoboRumble é amplamente usado pela comunidade para criar rankings atualizados de robôs nas competições 1-vs-1, Melee, Team e Twin Dual.

Em 2012, o Robocoder Julian ("Skilgannon") criou o LiteRumble, um sistema RoboRumble leve, fácil de implantar, projetado para rodar no Google App Engine.

Em maio de 2010, um plugin .Net foi disponibilizado para o Robocode usando uma ponte .NET/Java, permitindo desenvolver robôs para o .Net Framework 3.5 além do Java. Esse recurso foi fornecido por Pavel Savara, um grande colaborador do Robocode.

Em abril de 2021, o plugin .Net foi descontinuado devido a problemas sérios no .Net Framework e na cadeia de ferramentas necessária para construir o plugin e sua documentação.

REQUISITOS DO SISTEMA

Para rodar o Robocode, o Java 8 ou uma versão mais recente deve estar instalado no seu sistema. Tanto o Java Runtime Environment (JRE) quanto o Java Developer Kit (JDK) podem ser usados. Note que o JRE não inclui o compilador padrão do Java (javac), que vem com o JDK. No entanto, o Robocode possui um compilador embutido ECJ (Eclipse Compiler for Java). Assim, é suficiente rodar o Robocode apenas com o JRE.

Também é importante que estas variáveis de ambiente estejam configuradas antes de rodar o Robocode:

  • JAVA_HOME deve apontar para o diretório do Java (JDK ou JRE).
    Exemplo no Windows: JAVA_HOME=C:\Program Files\AdoptOpenJDK\jdk-16.0.0.36-hotspot
    Exemplo no Linux e macOS: JAVA_HOME=/usr/lib/jvm/adoptopenjdk-16-hotspot-amd64

  • PATH deve incluir o caminho para o bin do diretório do Java (JAVA_HOME) que contém o executável java para iniciar a JVM.
    Exemplo no Windows: PATH=%PATH%;%JAVA_HOME%
    Exemplo no Linux/macOS: PATH=${PATH}:${JAVA_HOME}/bin

API DO ROBOCODE

A API do Robocode está disponível aqui:

  • Java Robot API
  • Java Control API

A API do Robocode consiste em várias APIs.

  • Robot API: No pacote Java robocode. Usada para desenvolver robôs e é a única parte da API que os robôs podem acessar.

  • Robot Interfaces: No pacote Java robocode.robotinterfaces. Usada para desenvolver novos tipos de robôs com uma API diferente da padrão. Nota: As regras do jogo e comportamentos dos robôs não podem ser alterados.

  • Control API: No pacote Java robocode.control. Usada para permitir que outro aplicativo inicie batalhas com robôs selecionados no Robocode e recupere os resultados. Também é possível obter snapshots dos robôs e balas (posição, direção, energia, etc.) em um momento específico da batalha.

ENTRE NA COMPETIÇÃO

Se quiser desafiar seu(s) robô(s) e a si mesmo como Robocoder, o LiteRumble é a melhor maneira. O LiteRumble é um esforço colaborativo para manter um ranking atualizado dos bots do Robocode.

Então não hesite em entrar na competição RoboRumble.

LINHA DE COMANDO

É possível especificar opções e propriedades predefinidas pela linha de comando ao rodar o Robocode. O uso pode ser listado digitando no prompt ou terminal:

robocode -help

Por exemplo, é possível:

  • Desabilitar a interface gráfica (GUI).
  • Desabilitar a segurança específica do Robocode (mas não a do JVM).
  • Ativar/desativar o modo de depuração, útil para depurar robôs.
  • Executar uma batalha baseada em um arquivo .battle existente.
  • Reproduzir uma batalha gravada visualmente.
  • Salvar os resultados das batalhas em um arquivo CSV.

RELATAR PROBLEMAS / BUGS

Se você encontrar um problema no Robocode, é recomendado relatá-lo assim que possível. Quanto antes, melhor.

Um bug deve ser relatado na página de Bugs do SourceForge para o Robocode. Cada bug será priorizado de acordo com seu impacto no jogo.

Ajuda muito se você descrever os passos necessários para reproduzir o problema. Você pode fornecer uma captura de tela, código-fonte ou qualquer coisa que mostre o bug. Também é bom informar qual sistema operacional, versão do Robocode e do Java você está usando.

Se você for um usuário registrado no SourceForge (registre-se aqui), poderá adicionar um "monitor" ao seu relatório de bug. Assim, você receberá notificações quando alguém comentar no seu relatório e poderá acompanhar o status do bug, por exemplo, quando ele for corrigido e em qual versão do Robocode a correção estará disponível.

Se você for desenvolvedor e souber como corrigir o bug, pode fazer isso enviando um pull request no GitHub. Ao corrigir o bug, você se torna um colaborador do Robocode. Saiba mais sobre como contribuir aqui. Aceitamos correções sob os termos da EPL.

SOLICITAÇÕES DE FUNCIONALIDADES

Se você tem uma ideia para uma nova funcionalidade ou melhoria para o Robocode, compartilhe sua ideia enviando uma solicitação ou iniciando uma discussão no grupo Robocode Application Developers.

Uma solicitação de funcionalidade deve ser feita em Feature Requests no SourceForge para o Robocode. Cada solicitação será priorizada entre as demais.

Note que se a funcionalidade for uma grande mudança no jogo, como alterar o comportamento do robô, pode não ser aceita, pois o Robocode é usado em competições como o LiteRumble.

Ajuda muito se você descrever sua ideia em detalhes e como acha que ela pode ser implementada no Robocode. Por exemplo, será possível estender uma funcionalidade existente?

Se você for um usuário registrado no SourceForge (registre-se aqui), poderá adicionar um "monitor" à sua solicitação. Assim, receberá notificações quando alguém comentar e poderá acompanhar o status, por exemplo, quando a funcionalidade for implementada e em qual versão estará disponível.

Se você for desenvolvedor e souber como implementar a funcionalidade, pode fazê-lo se ela for aceita. Ao implementar, você se torna um colaborador do Robocode. Saiba mais sobre como contribuir aqui. Aceitamos implementações sob os termos da EPL.

COMO CONTRIBUIR

Se você quer contribuir com o Robocode, por exemplo, com uma nova funcionalidade ou correção de bug, comece lendo o Guia do Desenvolvedor para construir o Robocode.

Aceitamos mudanças de código sob os termos da EPL.

Existe pouca ou nenhuma documentação sobre o funcionamento interno do Robocode, então o código-fonte precisa ser examinado para entender como ele é implementado. Por isso, é necessário ser um desenvolvedor Java experiente para entender a estrutura do Robocode.

O Robocode é dividido em vários módulos. Você pode ler o post do Pavel Savara para ter uma boa visão geral aqui:

  • Módulos do Robocode

Ajuda sobre o funcionamento interno do Robocode pode ser obtida no grupo Robocode Application Developers, onde você pode se registrar e iniciar um novo tópico. Essa é a melhor forma de obter informações e tirar dúvidas sobre detalhes internos do Robocode.

Se a contribuição for uma mudança pequena (menos de 10 arquivos), a forma preferida é fazer um pull request no GitHub.

Seu pull request será revisado e testado antes de ser aceito e incorporado ao Robocode. Também pode ser feito trabalho adicional por outros desenvolvedores para finalizar ou ajustar o que foi enviado.
